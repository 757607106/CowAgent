## Design Context

### Users
企业团队中的运营、客服、实施和开发人员通过 Web 控制台管理数字员工。
他们需要快速查看 Agent 状态、配置能力、接入渠道，并在同一工作流中检查记忆与知识资产。

### Brand Personality
专业、稳健、可控。
界面需要传达可信赖、清晰有序、适合长期运营维护的控制台体验。

### Aesthetic Direction
延续当前浅色控制台基调，采用更聚合的信息布局和更明确的层级分区。
避免炫技式装饰，强调结构清晰、状态可感知、重点操作收束到 Agent 详情工作台。

### Design Principles
1. 以 Agent 为中心组织配置、能力、记忆和知识，减少页面跳转。
2. 保持企业级控制台的稳定感，优先清晰层级和可读性。
3. 让“查看状态”“修改配置”“刷新内容”在同一视图内完成。
4. 对隔离边界表达明确，记忆和知识只展示当前 Agent 上下文。
5. 复用现有 Ant Design / Ant Design X 能力，避免重复实现。
