# CowAgent Web 控制台页面功能梳理

> 用途：供 Figma 基于现有前端功能完整设计 UI 页面。本文档只整理功能与状态，不包含 UI 改版实现。
>
> 源码范围：`channel/web/ui/src/App.tsx`、`channel/web/ui/src/pages/*`、`channel/web/ui/src/chat/*`、`channel/web/ui/src/services/api.ts`、`channel/web/ui/src/types.ts`
>
> 当前版本标识：CowAgent 2.0.6

## 1. 全局应用框架

### 1.1 路由与导航

应用使用 HashRouter，浏览器地址形如 `/chat#/usage` 或 `/chat#/chat`。主导航菜单包含：

| 菜单 | 路由 | 页面 |
| --- | --- | --- |
| 对话 | `/chat` | ChatPage |
| 配置 | `/config` | ConfigPage |
| AI 员工 | `/agents` | AgentsPage |
| 技能 | `/skills` | SkillsPage |
| MCP | `/mcp` | McpPage |
| 渠道接入 | `/channels` | ChannelAccessPage，默认渠道 Tab |
| 用量 | `/usage` | UsagePage |
| 租户 | `/tenants` | TenantsPage |
| 租户成员 | `/tenant-users` | TenantUsersPage |
| 任务 | `/tasks` | TasksPage |
| 日志 | `/logs` | LogsPage |

历史路由和重定向：

| 路由 | 结果 |
| --- | --- |
| `/` | 重定向到 `/chat` |
| `/tools` | 重定向到 `/agents` |
| `/memory` | 重定向到 `/agents` |
| `/knowledge` | 重定向到 `/agents` |
| `/bindings` | 打开渠道接入页，默认绑定 Tab |
| `*` | 重定向到 `/chat` |

### 1.2 应用壳层

桌面端：

- 左侧固定 Sider，宽度 220。
- 顶部品牌区：`CowAgent 2.0.6`。
- 登录后展示当前租户/团队 Tag、当前用户名称或账号、退出登录按钮。
- 下方主菜单按页面路由高亮。
- 右侧主内容区承载页面。

移动端：

- 顶部移动 Header：菜单按钮、CowAgent 标识、当前页面名、当前团队 Tag。
- 点击菜单按钮打开左侧 Drawer，Drawer 内复用桌面菜单与登录信息。

### 1.3 认证与登录

启动时调用 `/api/auth/check` 判断认证状态。

状态：

- 检查中：全屏 Spin。
- 不需要认证或已认证：进入控制台。
- 需要认证且未登录：进入登录/注册页。

登录页功能：

- 标题：CowAgent 控制台。
- Tab：登录。
- 租户认证模式下需要输入登录账号和密码。
- 非租户认证模式下只输入密码。
- 支持回车提交。

注册页功能：

- 仅租户认证模式显示。
- 当后端返回 `bootstrap_required` 时默认打开注册 Tab。
- 字段：团队名称、登录账号、姓名、密码、确认密码。
- 密码最少 8 位。
- 校验两次密码一致。
- 注册成功后自动登录。

退出登录：

- 点击侧边栏退出按钮调用 `/api/auth/logout`。
- 成功后重新检查认证状态。

### 1.4 全局运行范围 Runtime Scope

全局上下文包含：

- `tenantId`：当前租户，登录态来自 auth user，否则默认 `default`。
- `agentId`：当前运行 Agent，持久化到 `localStorage.cowagent_runtime_agent_id`。
- `bindingId`：当前渠道绑定，当前前端启动时会清空。
- `agentOptions`：运行时 Agent 下拉选项，通过 `listAgentsSimple` 加载。
- `setAgentScope`：切换当前对话 Agent，同时清空 binding。
- `refreshAgentOptions`：刷新 Agent 选项，用于创建/更新 Agent 后同步下拉。

设计注意：

- 全局需要支持“团队/用户信息”“当前 Agent 上下文”“退出登录”“移动端菜单”。
- 所有页面应考虑 loading、empty、error message、destructive confirm 等基础状态。

## 2. 对话页 `/chat`

### 2.1 页面目标

面向浏览器 Web 的核心聊天工作台。用户在当前租户、当前 Agent 作用域下发起多轮对话、上传附件、查看流式回复、工具调用、深度思考和历史会话。

### 2.2 页面结构

主要分为两栏：

- 左侧会话栏。
- 右侧聊天区域。

### 2.3 会话栏功能

会话来源：

- 调用 `listSessions(scope, 1, 200)` 获取最近 200 条会话。
- 会话按时间分组：今天、昨天、更早。
- 当前会话 ID 持久化到本地存储，按租户/Agent/Binding 作用域隔离。

会话项展示：

- 会话标题。
- 更新时间。
- 消息数量。

会话操作：

- 新建会话：生成新 session id，清空消息区并设为当前会话。
- 刷新会话列表。
- 点击会话切换当前会话。
- 删除会话：带确认操作。
- 当前会话高亮。

历史消息：

- 调用 `history(scope, sessionId, 1, 200)`。
- 只展示最近 200 条。
- 如果后端返回更多历史，页面提示“当前展示最近 200 条消息”。
- 支持上下文清理分割线：`— 以上内容已从上下文中移除 —`。

空态：

- 无会话时仍可新建会话。
- 当前 session 无消息时显示欢迎提示和快捷 Prompt。

### 2.4 聊天消息区

消息角色：

- User：用户输入、附件展示。
- Assistant：AI 回复、流式内容、工具步骤、媒体结果。
- System：系统消息。
- Divider：上下文清理分割线。

用户消息：

- 展示文本 Markdown。
- 展示附件列表，附件类型包括 image、video、file。

AI 消息：

- 使用 ThoughtChain 展示运行步骤。
- 支持深度思考步骤：标题“深度思考”，可折叠，显示耗时。
- 支持工具调用步骤：
  - 工具名。
  - Input 面板。
  - Output 或 Error 面板。
  - 长文本折叠/展开。
  - 输入输出可复制。
  - 执行中、成功、失败、中止状态。
- 支持阶段进展/阶段回答。
- 支持主回答 Markdown。
- 支持引用来源折叠列表，最多提取 8 个 HTTP/HTTPS 来源。
- 支持 Markdown 代码高亮。
- Mermaid 代码块在流式完成后渲染图表。
- 视频链接可渲染为播放器。
- 支持 AI 返回 image、video、file 媒体。
- 成功回复底部提供复制回答按钮。

流式状态：

- 初始显示“生成中…”。
- SSE 事件包括 reasoning、tool_start、tool_end、phase、delta、text、message_end、image、video、file、cancelled、error、done。
- 用户可以中止当前回复。
- 中止后展示“已停止回复”状态。
- 错误后展示“回复失败”状态。

### 2.5 输入区 Sender

基础输入：

- 多行输入框。
- Placeholder 随状态变化。
- autosize，最少 2 行，最多 6 行。
- 发送按钮。
- 流式生成中展示停止按钮。

快捷命令：

- 输入 `/` 时显示命令建议。
- 支持选择命令填入输入框。
- 当前命令清单：
  - `/help`
  - `/status`
  - `/context`
  - `/context clear`
  - `/skill list`
  - `/skill list --remote`
  - `/skill search `
  - `/skill install `
  - `/skill uninstall `
  - `/skill info `
  - `/skill enable `
  - `/skill disable `
  - `/memory dream `
  - `/knowledge`
  - `/knowledge list`
  - `/knowledge on`
  - `/knowledge off`
  - `/config`
  - `/logs`
  - `/version`

输入历史：

- 本地保留最近 50 条输入。
- 在非多行、非命令建议状态下，上/下方向键切换历史输入。

附件：

- 支持点击附件按钮展开上传面板。
- 支持选择多个附件。
- 支持粘贴文件上传。
- 支持拖拽文件上传。
- 上传接口：`/api/upload`。
- 上传成功后显示附件列表。
- 支持删除待发送附件。
- 发送后清空附件。
- 输入区展示“已选 N 个附件”状态。

深度思考：

- 页面初始化从配置读取 `enable_thinking`。
- Sender footer 提供开关。
- 每次请求带上 `enable_thinking`。

Agent 切换：

- Sender footer 提供当前 Agent 下拉。
- 选项来自全局 `agentOptions`。
- 切换后刷新当前 runtime scope。

### 2.6 快捷 Prompt 空态

无消息时展示欢迎区和快捷 Prompt：

- 介绍你当前可用的能力。
- 梳理一下这个项目的核心模块。
- 列出当前工作区可用的工具链。

点击 Prompt 直接提交。

### 2.7 Figma 必画状态

- 会话列表正常/空/加载。
- 会话分组和当前选中态。
- 新建、刷新、删除确认。
- 聊天空态。
- 用户文本消息、用户附件消息。
- AI 流式生成中。
- AI 深度思考折叠/展开。
- AI 工具调用执行中/成功/失败。
- AI Markdown、代码块、Mermaid、引用来源、媒体附件。
- 输入区默认、上传面板展开、拖拽中、上传中、生成中停止态。
- 命令建议浮层。
- 移动端聊天布局。

## 3. 配置页 `/config`

### 3.1 页面目标

管理系统模型接入、运行策略和安全密码。该页是系统级配置，不是单个 Agent 配置。

### 3.2 页面顶部

标题：系统配置。

操作：

- 刷新。
- 保存全部更改。
- 当没有未保存更改时，保存按钮禁用。

概览卡片：

- 当前供应商。
- 当前模型。
- API Key 状态：已配置、待填写或无需。
- 自定义模型状态。
- 页面状态：有未保存更改或已同步。
- 安全状态。

### 3.3 模型接入模块

模块内分为 3 步：

Step 01：供应商与模型

- 供应商 Select。
- 标准模型 Select。
- 如果启用自定义模型或无供应商时，标准模型选择禁用。

Step 02：凭证

- API Base 输入框。
- API Key 密码输入框。
- 已保存 API Key 会以掩码展示。
- 聚焦掩码 Key 时清空，避免直接覆盖。
- 如果掩码未被改动，保存时不覆盖原 Key。

Step 03：自定义模型

- 开关：使用自定义模型名称。
- 自定义模型名输入框。
- 状态 Alert。

操作：

- 重置本区改动。
- 保存模型接入。

保存校验：

- 校验 provider 和 model。
- 保存 `model`、`use_linkai`、`bot_type`。
- API Base 有值才保存。
- API Key 只有非掩码且有值时才保存。

### 3.4 运行策略模块

预设：

- 稳健默认。
- 平衡模式。
- 高推理模式。

字段：

- `enable_thinking`：是否启用深度思考。
- `agent_max_context_tokens`：最大上下文 token，最小 1024，步进 1024。
- `agent_max_context_turns`：最大上下文轮数，范围 1 到 200。
- `agent_max_steps`：最大执行步骤，范围 1 到 200。

操作：

- 恢复平衡模式。
- 保存运行策略。

### 3.5 安全模块

显示条件：

- 仅非租户认证模式显示。

功能：

- 设置或更新控制台访问密码。
- 已有密码使用掩码。
- 清空密码并保存表示移除密码。
- 设置新密码成功后页面重新加载。

### 3.6 未保存状态

底部 sticky bar：

- 仅存在未保存改动时显示。
- 展示修改摘要。
- 操作：取消修改、保存全部更改。

### 3.7 Figma 必画状态

- 配置已同步。
- 有未保存改动。
- 模型供应商/标准模型/自定义模型三种组合。
- API Key 已配置掩码、待填写、无需。
- 运行策略预设选择。
- 安全密码模块显示/隐藏。
- 底部 sticky 保存条。

## 4. AI 员工页 `/agents`

### 4.1 页面目标

管理租户下的 AI 员工/Agent，包括创建、配置模型、工具、技能、MCP、知识库、记忆与发起会话。

### 4.2 AI 员工大厅

顶部工具栏：

- 视图切换：卡片 / 表格。
- 租户选择。
- 搜索输入：按名称、ID、模型筛选。
- 刷新。
- 新员工入职。

概览统计：

- 在岗员工。
- 可独立服务。
- 带知识库。
- 能力连接。

空态：

- 没有员工时提示创建。
- 搜索无结果时提示清空或调整关键词。

### 4.3 卡片视图

每个 Agent 卡片展示：

- 员工头像/封面。
- 名称。
- 状态 Badge。
- Agent ID。
- 模型。
- 系统提示摘要。
- 标签：
  - 官方认证。
  - 知识库/通用行业。
  - AI 助手/基础对话。
  - 外部服务或模型状态。
- 能力计数。
- 工作线信息。

操作：

- 配置。
- 应用：进入对话并切换当前 Agent。

### 4.4 表格视图

列：

- 员工：头像、名称、状态、ID。
- 模型。
- 能力：工具、技能、MCP 启用数量。
- 知识库。
- 租户。
- 操作：配置、删除。

删除限制：

- 默认 Agent 不允许删除。

### 4.5 新员工入职弹窗

字段：

- 租户：如果只有一个租户可隐藏或默认。
- Agent ID：可选。
- 名称：必填。
- 模型：必填，可搜索/标签模式。
- 系统提示词。
- 工具：多选。
- 技能：多选。
- 知识库启用开关。

提交：

- 创建 Agent。
- 刷新 Agent 列表。
- 刷新全局运行时 Agent 下拉。

### 4.6 Agent 详情配置页

进入方式：

- 点击卡片或表格中的“配置”。

顶部 Hero：

- 返回大厅。
- 员工头像、名称、状态。
- 租户/Agent ID。
- 发起会话。
- 保存变更。

Tab 1：核心配置

- 名称。
- 模型。
- 系统提示词。
- 是否启用知识库。
- 保存核心配置。

Tab 2：外接能力

- 工具能力：
  - 工具复选框列表。
  - 计数。
  - 空态。
- 技能能力：
  - 技能复选框列表。
  - 计数。
  - 空态。
- MCP 连接：
  - 嵌入 MCP 管理面板。
  - 支持新增、编辑、测试、删除 MCP Server。
- 默认 Agent 特殊规则：
  - 工具/技能为空表示继承全部。

Tab 3：长期记忆

- 子 Tab：记忆与梦境。
- 子 Tab：知识库。
- 两个子面板都按当前 Agent scope 加载。

操作：

- 发起会话：切换全局 Agent 并跳转 `/chat`。
- 保存变更：保存当前表单与能力配置。
- 删除：非默认 Agent 可删除。

### 4.7 Figma 必画状态

- 大厅卡片视图和表格视图。
- 搜索/租户筛选/无结果。
- 创建 Agent 弹窗。
- Agent 详情三大 Tab。
- 默认 Agent 的禁删和继承提示。
- 工具/技能/MCP 空态。
- 进入对话动作。

## 5. 技能页 `/skills`

### 5.1 页面目标

管理当前运行作用域下可调用的技能，控制启用状态，区分系统内置技能和本地技能。

### 5.2 页面顶部

标题：技能库。

操作：

- 视图切换：卡片 / 表格。
- 刷新。

统计卡片：

- 全部技能。
- 已启用。
- 系统内置。
- 本地技能。

筛选工具栏：

- 搜索：技能名称、真实 name、描述、分类。
- 状态筛选：全部、已启用、已停用。
- 来源筛选：全部、系统内置、本地技能。

排序规则：

- 已启用优先。
- 系统内置优先。
- 名称字母排序。

### 5.3 卡片视图

每张技能卡展示：

- 图标：系统内置使用认证图标，本地技能使用代码图标。
- 展示名。
- 技能真实 name。
- 状态 Badge：已启用/已停用。
- 来源 Tag。
- 分类 Tag。
- 版本 Tag。
- 描述，最多 3 行。
- 底部来源说明。

操作：

- Switch 启用/禁用。
- 删除按钮。
- 系统内置技能删除按钮禁用，并有 Tooltip。
- 本地技能删除前 Popconfirm。

### 5.4 表格视图

列：

- 技能：图标、展示名、真实 name。
- 来源。
- 分类。
- 描述。
- 状态。
- 操作：启停、删除。

表格行为：

- pageSize 10。
- 横向滚动。

### 5.5 Figma 必画状态

- 统计卡片。
- 筛选工具栏。
- 卡片视图。
- 表格视图。
- 启用/禁用 loading。
- 系统内置不可删。
- 本地技能删除确认。
- 无技能/无搜索结果空态。

## 6. MCP 页 `/mcp`

### 6.1 页面目标

按智能体管理 MCP Server，支持创建、编辑、删除、测试连接、查看运行中工具和输入 Schema。

### 6.2 顶部区域

标题：MCP 服务管理。

操作：

- Agent 选择器。
- 刷新。
- 新增服务器。

状态：

- Agent 列表加载中。
- Agent 列表加载失败，展示 Alert 和重试。
- 无可管理 Agent，展示 Empty。

### 6.3 Agent 摘要

展示：

- Agent 头像。
- Agent 名称。
- Agent ID。
- 模型。
- 工具数量。
- 技能数量。
- MCP Server 数量。

说明：

- 新增、编辑、删除 MCP Server 会直接写回所选 Agent 的 `mcp_servers` 配置。
- 保存时保留原 Agent 的工具、技能、系统提示、知识库等字段。

### 6.4 MCP Server 列表

每个 Server 卡片展示：

- Server 名称。
- 启动命令 command。
- 参数 args。
- 环境变量数量。

操作：

- 查看工具 / 收起工具。
- 测试连接。
- 编辑。
- 删除。

工具展开区：

- 点击“查看工具”后懒加载工具列表。
- 操作：重新发现。
- 状态：加载中、加载失败、无工具、工具列表。
- 工具卡展示工具名、描述、查看输入 Schema。
- 点击工具打开 Schema 弹窗。

### 6.5 新增/编辑 MCP Server 弹窗

字段：

- 服务器名称：必填；编辑时禁用。
